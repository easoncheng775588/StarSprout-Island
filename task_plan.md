## Decisions

- 新项目使用独立目录 `k12-learning-game/`
- 前端优先使用 React + Vite + TypeScript
- 后端优先使用 Spring Boot 3 + Java 17
- 数据库生产目标为 MySQL，测试与本地单测优先使用 H2
- MVP 首批实现主世界首页、学科地图、关卡详情和学习记录提交
- Maven 本地仓库固定在工作区 `.cache/m2`
- npm 缓存固定在工作区 `.cache/npm`
- 当前前端已切换为真实 API 取数，保留测试用 fetch mock
- 家长中心与排行榜页面先用页面级测试保障，后续可再补 E2E
- 首页现在提供家长中心与排行榜显式入口
- 后端默认使用 H2 + `data.sql` 种子数据，生产可切换 `mysql` profile
- 本轮开始支持后端 `activityConfigJson` 驱动玩法配置，`LevelPlayer` 优先读取后端配置，缺失或非法时回退到本地配置
- 先把课程模型升级为“按孩子学段过滤章节/关卡”，避免一年级到四年级内容直接混在同一张地图里
- 每个新学段先做 MVP 粒度：每科 1-2 个主题章节、每章 2-4 个关卡、优先复用现有互动玩法
- 幼小衔接内容继续保留，作为 `幼小衔接` 学段课程；一年级到四年级按相同结构继续扩充
- 家长端新增阶段报告、知识点掌握图谱和错题复习闭环，作为后续完整学情报告与薄弱点闭环的底座
- 每日任务列表仍基于现有关卡完成记录动态生成，但领取奖励记录需要持久化，避免重复领取
- 错题本列表基于错题记录聚合生成，复习提交结果需要持久化，用最近一次复习结果决定是否移出错题本
- 通关后的即时反馈优先放在原奖励卡内，不新增额外请求；后端在完成关卡响应中直接返回本周星星榜反馈
- 首页今日任务状态先基于现有 `nextLevelCode` 前端推导，不新增后端字段；后续若任务规则复杂化，再接入每日任务 API 摘要
- 学习路径锁定规则采用“第一个未完成为推荐，其后 locked”的简单主线机制，适合当前 MVP 内容节奏
- 题库配置中心先做只读目录，展示已经下沉到后端的玩法配置质量，后续再演进为可编辑 CMS
- 家长端多孩子对比不做压力型排名，当前采用“当前查看置顶 + 其余按本周星星/完成关数排序”的温和展示方式
- 动画解读先用前端 CSS 动画和分镜卡实现“视频化讲解”MVP，后续可把同一数据结构替换为真实 mp4/素材库资源
- 幼小衔接继续增强时优先补认知覆盖缺口，而不是重复已有的数数、拼音和字母分组内容
- 逐年级产品化迭代不得盲目加表；课程内容继续优先复用 `subjects`、`chapters`、`levels`、`level_steps`，业务引用使用稳定字符串 code
- 前端关卡播放器继续扩内容前必须先抽复用组件，避免 `LevelPlayer.tsx` 成为不可维护的巨型页面

## Errors Encountered

| Error | Attempt | Resolution |
| --- | --- | --- |
| `mvn -v` 在 login shell 下回包异常慢 | 1 | 改用 `login=false` 运行环境检查 |
| Maven 默认写入 `~/.m2` 被拒绝 | 1 | 改为 `-Dmaven.repo.local=/Users/easoncheng/Documents/New project/.cache/m2` |
| Maven 下载依赖时 DNS 失败 | 2 | 通过允许外网下载依赖解决 |
| npm 安装长时间无结果 | 1 | 通过允许外网下载依赖解决 |
| Vitest 未注入全局 `describe/test` | 1 | 在 `vite.config.ts` 中启用 `test.globals` |
| Mockito inline mock maker 在当前 JDK 环境失败 | 1 | 切换到 `mock-maker-subclass` |
| subagent 调用供应商 503 熔断 | 1 | 停止重试，改为主线程继续实现 |
| MySQL 驱动依赖下载受沙箱网络限制 | 1 | 申请联网下载 `mysql-connector-j` 后继续测试 |
| 当前课程内容未按孩子 `stageLabel` 过滤 | 1 | 本轮先补学段字段与过滤逻辑，再扩一年级到四年级内容 |
| GitHub 推送目标不可用 | 1 | 当前 git 仓库未配置 remote，先完成本地提交，待配置远端后再推送 |
| subagent 二次调用供应商 503 熔断 | 2 | 按用户要求继续项目，降级为主线程直接实现二年级 MVP |
| 知识点图谱卡片缺少稳定 key | 1 | 后端补充 `knowledgePointCode`，前端使用该字段作为 React key 并补充回归测试 |
| 代码审查 subagent 因额度限制失败 | 1 | 不阻塞提交，改为主线程复核契约并以全量前后端测试兜底 |
