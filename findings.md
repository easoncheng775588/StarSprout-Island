## Interactive Level Rendering

- 当前前端没有扩后端 step payload 协议，而是在 `LevelPlayer` 内按已知 `levelCode + stepId` 做玩法配置映射
- 已覆盖的玩法包括：数字拖拽计数、数字选择、汉字识别、拼音听辨、字母跟读、英语单词配对

## Productization Improvements 2026-04-20

- 后端玩法配置已开始落地到 `level_steps.activityConfigJson`，代表关卡可直接返回前端可解析的 `kind` 配置
- 前端 `LevelPlayer` 现在优先读取后端玩法配置，非法或缺失时回退本地配置，适合后续逐步迁移到题库/CMS
- 每个 step 可携带知识点 code/title 与题库变体数，前端可展示“知识点、题库变体、素材风格、音频质量”提示
- 家长端已从概览面板扩展为阶段报告、知识点掌握图谱、错题复习闭环三类数据
- 知识点图谱需要稳定标识；已补 `knowledgePointCode` 并让前端用它作为 React key，避免同名知识点卡片复用错位
- 当前素材/音频仍主要是配置提示与浏览器 TTS/现有视觉风格，后续若追求正式商业化体验，应继续引入真实图片素材库、儿童友好配音资源与可运营题库后台

## Engagement Loop 2026-04-20

- 每日任务可由现有学习记录动态生成，当前不必先做任务表；MVP 阶段更适合用主线下一关、错题复习、星星目标三类任务验证留存价值
- 错题本可以复用 `LevelCompletionEntity.wrongCount` 聚合，先形成“错题点 -> 复习步骤 -> 回到关卡”的闭环
- 学习路径锁定已统一到后端状态，学科地图和学习路径页都能使用 `completed/recommended/locked`
- 题库配置中心目前是只读目录，能帮助运营/开发识别哪些关卡已完成后端配置下沉，后续可扩展编辑、上下架、素材上传和音频审核
- 当前错题掌握状态仍由错题次数粗略判断，后续建议加入按知识点的多次复习记录与“连续答对后掌握”规则

## P0 Settlement Loop 2026-04-20

- 每日任务列表适合继续动态生成，但奖励领取必须有持久化流水；否则孩子可通过刷新或重复请求重复获得星星
- 当前实现按 `child + taskCode + taskDate` 记录领取状态，可满足 MVP 的每日一次奖励结算
- 错题掌握不宜只靠历史错题次数判断；新增复习提交流水后，可以用“最近一次复习结果”覆盖错题本展示状态
- 当前掌握规则先采用 `correctCount >= 3 && wrongCount == 0`，便于孩子端一键复习闭环；后续可演进为按知识点连续多次正确、间隔复习和遗忘曲线
- P1 更适合沿现有体验面继续增强：通关庆祝与榜单反馈、家长端多孩子对比、首页今日任务状态化

## P1 Celebration Loop 2026-04-20

- 通关后的“下一步动力”最适合复用现有奖励卡，而不是跳转到单独页面；孩子能在同一屏看到星星、徽章、下一关和榜单反馈
- 榜单反馈放进 `CompleteLevelResponse` 可以避免前端完成关卡后再请求一次排行榜，交互更轻也更稳定
- 当前排名反馈沿用本周星星榜规则，返回 `rankBefore/rankAfter/trendLabel/message`，前端只负责展示，不在浏览器端自行推算名次
- 旧响应兼容仍然必要；前端为 `leaderboardFeedback` 准备了兜底值，避免本地联调时前后端版本不同步导致通关页崩溃

## P1 Home Retention 2026-04-20

- 首页今日任务不需要先等待后端新增状态字段；当前 `nextLevelCode` 已经能表达“还有主线可推进”或“主线已完成”
- 状态化任务卡比纯文案更适合儿童入口：孩子能快速看懂今天是继续挑战，还是去学习路径复盘
- 连续学习提示放在任务卡中，比只放统计数字更有情绪反馈，更接近正式产品的留存入口

## P1 Parent Multi-Child Comparison 2026-04-21

- 多孩子对比适合放在家长中心，而不是孩子端；它是陪伴决策工具，不应变成孩子之间的压力来源
- 对比维度先选完成关数、本周星星、平均准确率和学段，能覆盖“进度、投入、稳定性、阶段”四类家长最关心的问题
- 展示顺序采用当前孩子置顶，其余孩子按本周星星和完成关数排序，比直接排名更温和，也更符合儿童学习产品的语气

## Preschool Content Gaps 2026-04-21

- 幼小衔接数学已有数数、20 以内加减、规律、比较、列式和故事题，但缺图形分类、方位理解、序数/第几个这类入学前空间与数感内容
- 幼小衔接语文已有常见字、拼音听辨和 3 个笔顺关，但缺偏旁观察、声调辨认、横竖撇捺等基础笔画巩固
- 幼小衔接英语已有 26 字母分组、开头音、生活词、颜色词和 3 个绘本，但缺大小写配对、家庭生活词和更多日常绘本场景
- 本轮适合新增 9 个小关卡，每科 3 个，复用现有互动玩法，避免先引入全新复杂玩法导致测试和内容配置膨胀

## Grade-by-Grade Productization 2026-04-22

- 当前最主要的代码质量风险在前端 `LevelPlayer.tsx`：多种玩法直接内联在页面中，继续扩关卡会让页面越来越难维护
- 课程数据库结构目前仍然够用，逐年级产品化不需要先新增内容表；新增关卡可以继续用稳定 code 写入 `levels` 和 `level_steps`
- `level_steps.id` 当前是技术主键，业务引用仍使用 `level_code + step_code`，后续内容不要依赖自增 ID 做业务含义
- 第一阶段最稳妥的工程顺序是：先抽可复用互动组件，再补幼小衔接成熟化内容，再推进一年级到四年级
- 成熟产品感不只来自关卡数量，还来自统一关卡流程、讲解、反馈、复习、家长端报告和成就闭环
- 音频控制在听辨、跟读、短句朗读中重复出现，适合先抽为 `AudioModeControls`，后续再继续拆分选项卡、反馈卡和媒体模型组件
- 幼小衔接数学补充“快速看数量、数的分合、看图加法”比单纯增加算式更符合入学准备阶段的数感目标
